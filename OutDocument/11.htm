Vulnerability Description
\n
Developers have tendency to log details in the development stage and track the functionality. They forget to sanitise the logs before App
goes into production. App logs can leak sensitive information such as user credentials, PIN, bank details, account no, URL, response
from
the server, cookie, phone numbers, contact details, and so on which can be used by an attacker to exploit the App.
\n
Example:
\n
\xe2\x80\xa2 The following example shows App leaking sensitive data through logs:
\xe2\x80\xa2 Log.i("MyActivity", "MyClass.getView() - getusername" + username)
\n
Instance(s)
\n
sg
\n
Recommendations
\n
\xe2\x80\xa2 Avoid logging sensitive information.
\xe2\x80\xa2 Sanitise the code by removing all sensitive loggers from the source code before the App goes into production.
